При получении потока символов из сокета, нужно ввести таймаут, потому, как сообщение модет быть фрагментировано.
Если при парсинге возникнет ошибка, буффер не стоит сбрасывать, пока не истечет таймаут сетвеого обмена.

Второй уровень абстракции обрабатывает фрагментацию самой команды. Потому, как одну RESP команду можно разбить
на отдельные части,  когда таких фрагментов будет достаточно для обработки - необходимо обработать этот пакет как цельный.
В этом случае, нужно не сбрасывать буфер до того момента, пока не попадется недопустимая последовательность для конкретной команжы.
То есть, если последовательность неполная - это допустимый случай, но если вдруг встречается последовательность,
даже неполная, которая недопустима в данной команде - можно сразу прерывать обработку с сообщением об ошибке, после того, как
вся последовательнотьсьть не убдет получена от клиента или не произойдет таймаут .

Пример:
C: *2\r\n
C: $4\r\n
C: LLEN\r\n
C: $6\r\n
C: mylist\r\n

S: :48293\r\n

*2\r\n$4\r\nLLEN\r\n$6\r\nmylist\r\n


На первой итерации, текщая позиция парснга не сохраняется, то есть результат парсинга не аккумулируется, а парсинг
заркскается заново после прибытия каждого нового блока данных. Предварительно, структура команд будет 
проверяется через проверку типов. Например, если команда представляет из себя массив, где эементами
являются целов, простая строка, и бинарная строка, то если какой-то из типов последовательности
не совпадает - команда ошибочна.

В дальнейшем, необходимо будет при поступлении блока данных парсить его, сохраная в выхожной буффер,
сырой буффер или усекается, оставляя в нем только остаток, неполный пакет, или же сохраняется указатель
 или тератор в буффере, на которм остановлен парсинг.




Первичная идея парсинга следующая. Для каждой команды описан массив типов, которой обеспечивает грубую проверку
корректности массива команд
template<class... Types>
struct type_array {
using as_tuple = std::tuple<Types...>;

    template<std::size_t I>
    using get = std::tuple_element_t<I, as_tuple>;

    static constexpr std::size_t size = sizeof...(Types);
};

using types = type_array<int, float>;
multiply<types::get<i>>();


Для каждого из RESP-типов должны быть описаны restrictions-constexpr типы. Напрмер, допустимые символы
или строки, опциональность поля и так далее. ЭТо обеспечивать вторую ступень провекри команды.
Третим этапом явялется следующий этап, непосредсвенно исполненипе команды.
